AIger E2E Testing Guide

This guide validates the current platform: prompt-to-workflow planning, inline log gates, marketplace installs, generated agents, workflow execution, HITL, A2A messages, reports, citations, and AIger Copilot.

1. Start Services

cd backend
python server.py
cd frontend
npm run dev

2. Builder Orchestrator

  1. Open /builder.
  2. Enter a realistic use case, such as contract risk review or repo modernization.
  3. Click Auto-build workflow.
  4. Confirm the top orchestrator button changes to the current step title.
  5. Open the log and confirm questions/install approvals are inside the log console as collapsible steps.
  6. After gates complete, confirm the final slide panel opens with one clean technical design preview.

3. Run Workflow

  1. Click Run workflow.
  2. Confirm the start animation appears while the run is created.
  3. On the run page, confirm graph edges animate during and after completion.
  4. Confirm execution estimates are compact text rows and A2A has most of the right-rail space.

4. Report and Sources

  1. Open the final report.
  2. Confirm report sections use Outcome Summary, Decision, Priority Actions, Key Findings, Structured Result, and Agent Evidence Trail.
  3. Confirm relevant risk/action rows use subtle red, amber, or green table tinting.
  4. Open Report Sources and confirm full source content renders readably with matched text highlighted.

5. AIger Copilot

  1. Open /tools-chat.
  2. Ask a platform or tool-using question.
  3. Confirm processing logs stream during response.
  4. Confirm Tool Activity is collapsed by default and scrollable after opening.
Real-data expectation: tests should use real installed marketplace agents, real uploaded files, real workflow runs, and real citations from the current workspace.
