Aigers Universe • Product and Engineering Guide

Platform Documentation

Aigers Universe is an enterprise AI platform for registering agents, composing workflows, grounding them with tools and documents, routing work through A2A, pausing for HITL approval, and observing every run through reports, traces, and live execution views.

38 seeded marketplace templates 19 tools and control actions 4 framework runtimes Projects and team sharing Workflow run controls AIger Copilot

Problem

Enterprises need more than single prompts. They need connected agents, evidence, approvals, runtime control, and auditability.

Objective

Provide one governed workspace for agents, workflows, tools, documents, projects, members, and execution visibility.

Core Model

Agents execute step by step, consume upstream context, call tools, emit A2A messages, and can pause for human approval.

1. Product Scope

Agent Lifecycle

  • Create custom agents with framework, prompt, tools, HITL, and A2A settings.
  • Install marketplace agents with recommended tools and models.
  • Export agents as LangGraph, LangChain, CrewAI, Agno, or Langflow-style artifacts.

Workflow Lifecycle

  • Build on ReactFlow with drag-drop node composition.
  • Override prompt, tools, A2A routing, and input bindings per node.
  • Run, pause, resume, stop, restart, and report from the workflow run page.

Grounding and Governance

  • Separate reusable KB context from workflow-scoped inputs.
  • Support MCP tools, docs search, web fetch, policy search, and remote A2A dispatch.
  • Use HITL for high-severity compliance and human approval flows.

2. Personas

AI Engineer

Builds agents, workflows, prompts, tool policies, and execution strategies.

Compliance Reviewer

Approves or rejects paused workflow decisions and reviews evidence for risky outputs.

Platform Admin

Oversees users, projects, templates, governance, and cross-workspace operations.

3. Main Product Surfaces

Surface Purpose Important Notes
Projects Create shared workspaces for workflows and runs. Supports member assignment and project-scoped visibility.
Marketplace Install pre-seeded templates. Install is idempotent for default installs.
Agents Register and manage custom agents. Includes local A2A card URLs and remote card validation.
Workflow Builder Compose a visual workflow. Supports Orchestrator AI and per-node overrides.
Workflow Run Observe and control execution. This is where pause, resume, stop, and start belong.
AIger Copilot Platform-aware chat and advisory workspace. Composer is for mode, model, tool, file, and send only. Workflow execution controls are intentionally not here.
HITL Human review queue. Approvals and rejections feed directly back into workflow execution.
Observability Charts and traces. Shows tokens, latency, cost, and recent execution traces.

4. Projects and Team Collaboration

Project Model

  • Projects group workflows under one workspace context.
  • Workflow definitions persist project_id.
  • Runs and visibility inherit project scope.

Team Member Flow

  • Project owners can add member emails from the Projects surface.
  • Members gain shared visibility into project workflows and related runs.
  • Admin users can oversee or delete projects from the admin surface.
Testing impact: project creation, member assignment, shared workflow visibility, and run access should be treated as first-class test scenarios, not optional edge cases.

5. Architecture

Frontend
  -> React 18 + Vite
  -> Routes for Dashboard, Projects, Marketplace, Agents, Builder, Runs, Copilot, HITL, Observability
  -> ReactFlow for builder and run visualization

Backend
  -> FastAPI routers
  -> MongoDB persistence
  -> Mongo vector retrieval
  -> FastMCP + FastAPI-MCP
  -> Workflow engine + framework runners

Execution
  -> Agent invokes tools
  -> Tool returns evidence or mutates state
  -> Trace persisted
  -> A2A message emitted
  -> Next step consumes upstream context
  -> Final report materialized

Frontend Stack

React 18, Vite, React Router, ReactFlow, Recharts, Tailwind CSS, lucide-react, sonner.

Backend Stack

FastAPI, Motor, MongoDB, FastMCP, FastAPI-MCP, LangGraph, LangChain, CrewAI, Agno, Mongo vector chunks, structlog, OpenAI-compatible gateway.

6. Runtime and State Lifecycle

  1. User creates or installs agents.
  2. User creates a project and optionally adds members.
  3. User builds a workflow manually or via Orchestrator AI.
  4. User attaches workflow inputs and or KB context.
  5. Backend creates a run and invokes framework-native agent runners step by step.
  6. Agents call tools through TOOL_REGISTRY and MCP-exposed definitions.
  7. Traces, A2A messages, outputs, and run status are persisted.
  8. If a tool triggers HITL, the run pauses until approval or rejection.
  9. Report generation materializes final structured and markdown outputs.

7. Workflow Run Controls

Pause

Available while the run is running or resuming. It queues a pause so the current agent can finish safely.

Resume

Available when the run is paused or another resumable terminal state. It restarts execution from the next valid step.

Stop

Available while running or paused. A paused run can be stopped immediately. A running run queues stop after the current step.

Start

The run page surfaces Start as the stopped-state resume label so users can clearly restart a stopped flow.

Product rule: workflow execution controls belong on the workflow run page, not in AIger Copilot. Copilot is a chat surface, not a workflow executor.

8. AIger Copilot

  • Supports AIger Copilot mode and General Reasoning mode.
  • Supports session history, rename, delete, grounded attachments, citations, and trace logs.
  • Supports explicit user-controlled tool selection including Auto tool.
  • Uses a compact composer for mode, model, tool, files, and send only.

11. Setup and Configuration

Backend

cd backend
python -m venv .venv
pip install -r requirements.txt
uvicorn server:app --reload --host 0.0.0.0 --port 8001

Key environment values include LLM_BASE_URL, LLM_API_KEY, MONGO_URL, DB_NAME, JWT_SECRET, and the workflow and chat input limits.

Frontend

cd frontend
npm install
npm run start

Use VITE_REACT_APP_BACKEND_URL to point at the backend origin.

12. 2D/3D Knowledge Graph and Chunking Runtime

Knowledge Graph UI

The platform now includes a dedicated Knowledge Graph page with:

  • 2D/3D rendering toggle using shared graph data
  • Main, sub, and chunk node layers
  • Similarity-weighted semantic links
  • Sector legend isolation filters
  • Node drag and persisted layout save

Chunking Strategies

Document ingestion supports four strategy options:

  • hierarchical
  • fixed-token
  • sentence-window
  • semantic

Each uploaded document stores the selected strategy in metadata as chunk_strategy.
