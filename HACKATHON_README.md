# AIger's Universe Hackathon Brief

AIger demonstrates a full enterprise agent workflow lifecycle:

1. User describes a use case.
2. Orchestrator asks targeted questions.
3. Orchestrator validates market signal and differentiators.
4. Orchestrator designs the technical solution and workflow.
5. Marketplace agents are installed through inline approval gates.
6. Missing custom capabilities are generated as agents.
7. The accepted plan builds a visual workflow.
8. A real run executes agents with MCP tools, A2A messages, HITL, and reports.

The strongest demo path is contract risk review or modernization planning because both show multi-agent reasoning, input grounding, approvals, and outcome reporting.

## Demo checklist

- Open Workflow Builder.
- Auto-build from a real prompt.
- Show inline questions/install gates inside the log console.
- Accept the final slide-panel plan.
- Run with a real document or repo input.
- Show animated execution edges, A2A messages, and HITL.
- Open the final report and source viewer.
- Ask AIger Copilot for platform guidance and open Tool Activity.
